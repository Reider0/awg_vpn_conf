
# 🛡️ VPN Dashboard (AmneziaWG + Telegram Bot)

Полноценная панель управления VPN-сервером внутри Telegram.
Проект использует **AmneziaWG** (протокол WireGuard с продвинутой обфускацией) для обхода блокировок (DPI), упакован в Docker с современной микросервисной архитектурой.

## ✨ Возможности

- **Управление через Telegram:** Создание, удаление пользователей, получение конфигурационных файлов и QR-кодов прямо в чате.
- **Защита трафика:** Встроенная обфускация AmneziaWG маскирует VPN-трафик под обычный мусорный трафик.
- **Авто-обновление:** "Умная" кнопка в боте, которая подтягивает свежий код с Git и безопасно пересобирает контейнеры без участия системного администратора.
- **Умные бэкапы:** Автоматическое сохранение базы данных и ключей в закрепленном сообщении Telegram.
- **Мониторинг:** Графики активности пиров и статус сервера в реальном времени.

## 🚀 Установка и настройка

### 1. Требования к серверу
- **ОС:** Рекомендуется чистый сервер (VPS) с **Ubuntu 24.04 LTS** (также поддерживаются 20.04/22.04 и Debian 11+).
- Telegram Bot Token (получить у [@BotFather](https://t.me/BotFather)).
- Ваш персональный Telegram ID (узнать у [@userinfobot](https://t.me/userinfobot)).
- GitHub Personal Access Token

### 2. Первичная подготовка сервера (Ubuntu 24.04)

Подключитесь к вашему серверу по SSH и выполните базовую настройку:

```bash
# 1. Обновляем списки пакетов и саму систему
sudo apt update && sudo apt upgrade -y

# 2. Устанавливаем базовые утилиты
sudo apt install curl git nano -y

# 3. Устанавливаем Docker (официальный установочный скрипт)
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh
rm get-docker.sh

# 4. Устанавливаем плагин Docker Compose
sudo apt update
sudo apt install docker-compose-plugin -y

# 5. Добавляем Docker в автозагрузку и запускаем
sudo systemctl enable docker
sudo systemctl start docker
```

### 3. Загрузка и установка проекта

Теперь, когда сервер готов, скачиваем и настраиваем VPN-бота:

```bash
# 1. Клонируем репозиторий в папку /opt
git clone https://github.com/YOUR_USERNAME/vpn-dashboard.git /opt/vpn-dashboard

# 2. Переходим в директорию проекта
cd /opt/vpn-dashboard

# 3. Запускаем скрипт установки
# Он выдаст нужные права файлам и настроит системного демона автообновлений
sudo bash install.sh
```

### 4. Настройка окружения (.env)

Скрипт установки автоматически создаст файл `.env`. Откройте его для редактирования:

```bash
nano .env
```

Заполните его вашими данными:

```ini
# --- Telegram ---
# Токен бота, полученный от BotFather
BOT_TOKEN=123456789:AAHgVy... 
# Ваш цифровой Telegram ID (никто другой не сможет пользоваться ботом)
ADMIN_ID=123456789

# --- Git (для работы авто-обновлений) ---
# Ссылка на ваш репозиторий (обязательно с .git на конце)
GIT_REPO=https://github.com/username/vpn-dashboard.git

# --- Авторизация Git (ТОЛЬКО ЕСЛИ РЕПОЗИТОРИЙ ПРИВАТНЫЙ) ---
# Если репозиторий публичный — оставьте эти два поля пустыми!
GIT_USERNAME=your_github_username
# Используйте Personal Access Token (PAT) с правами 'repo'
GIT_TOKEN=ghp_AbCdEf123456...
```

### 5. Финальный запуск

Соберите и запустите контейнеры:

```bash
docker compose up -d --build
```

**Готово!** Перейдите в Telegram и отправьте вашему боту команду `/start`.

## 📱 Клиентские приложения

Поскольку на сервере работает продвинутая защита от блокировок (обфускация), стандартный WireGuard клиент может не распознать конфигурацию. 

**Обязательно используйте официальные клиенты AmneziaWG:**

- **iOS:** [Скачать в AppStore](https://apps.apple.com/us/app/amneziawg/id6478942365) (Зеленая иконка с буквой W)
- **Android:** [Скачать в Google Play](https://play.google.com/store/apps/details?id=org.amnezia.awg)
- **Windows / macOS / Linux:** Используйте десктопный клиент [AmneziaVPN](https://amnezia.org/). При добавлении конфига выберите опцию импорта файла `.conf`.

*Примечание: Приложение легко считывает QR-коды, которые отправляет бот.*

## 🔄 Обновление системы

### Через Бота (Рекомендуемый метод)
1. В главном меню бота нажмите **"🆕 Проверить обновление"**.
2. Если версия на GitHub новее вашей, появится кнопка **"✅ Обновить сейчас"**.
3. Бот автоматически создаст резервную копию в закрепленном сообщении, отправит сигнал системному демону и уйдет в перезагрузку.
4. Сервер очистит кэш, скачает новый код и пересоберет контейнеры (~1-2 минуты). После этого бот снова станет доступен.

### Ручное обновление (через SSH)
Если необходимо обновиться вручную через консоль:

```bash
cd /opt/vpn-dashboard
sudo bash scripts/deploy.sh
```

## 🛠 Устранение неполадок

**1. При сканировании QR-кода выдает "Ошибку 900"**
Вы пытаетесь отсканировать текстовый QR-код через основное приложение *AmneziaVPN*. Установите легковесный клиент **AmneziaWG** (у него зеленая иконка) — он создан специально для этих целей.

**2. Бот пишет "Ошибка доступа Git" или "ошибка сети" при проверке обновлений**
Проверьте файл `.env`. Если ваш репозиторий приватный, вы обязаны указать `GIT_USERNAME` и `GIT_TOKEN`. Убедитесь, что токен имеет срок годности и права на чтение репозиториев (scope: `repo`).

**3. Как восстановить бэкап при переезде на новый сервер?**
1. Установите систему на новый сервер по инструкции выше.
2. В боте перейдите в меню восстановления и отправьте ему файл `backup_latest.tar.gz` (можно переслать прямо из закрепленного сообщения прошлого бота).
3. Бот распакует ключи, восстановит базу пользователей и **на лету** перезагрузит VPN-сервер. Все ваши старые клиенты сразу же смогут подключаться.

## 📜 Памятка для разработчика

Если вы вносите изменения в код и хотите выпустить новую версию:
1. Отредактируйте файл `VERSION` в корне проекта (например, измените `1.0.7` на `1.0.8`).
2. Закоммитьте изменения: `git add . && git commit -m "Update to 1.0.8"`.
3. Отправьте код в репозиторий: `git push`.
4. Зайдите в бота и нажмите кнопку обновления.
